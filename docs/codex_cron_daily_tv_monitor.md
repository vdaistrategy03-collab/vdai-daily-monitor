# Codex Cron Task Prompt

이 저장소에서 일간 TV 모니터링 리포트를 작성해줘. TV가 주력이지만, 전략적으로 의미 있는 모니터/프로젝터 신규 발표도 함께 포함해줘.

반드시 `AGENTS.md`를 따르고, 이 실행에서는 로컬 파일 생성까지만 수행해. Git 게시 작업은 wrapper 스크립트가 별도로 처리한다.

핵심 요구사항:

- 최신 `new_features/` 리포트들을 먼저 읽고 중복 보도를 피한다.
- 최신 리포트의 `기준 시각`을 확인해 그 시각부터 현재까지를 검색 구간으로 사용한다.
- 고정 24시간 창을 사용하지 않는다.
- 공식 채널과 주요 매체를 우선하고, 필요하면 지역 매체로 확장한다.
- Samsung, LG, Sony, TCL, Hisense, Panasonic, Philips/TP Vision, Sharp, Xiaomi, Amazon Fire TV, Google TV/Android TV, Roku, Apple TV를 우선 조사한다.
- TV 발표를 최우선으로 다루되, 모니터/프로젝터의 신제품, 펌웨어/UX, AI 기능, 게이밍/홈시네마 기능, 스마트 플랫폼 연동처럼 삼성 TV 경쟁력이나 인접 디스플레이 전략에 의미가 있는 신규 발표도 `신규 발표 확인 사항`에 함께 포함한다.
- 모니터/프로젝터는 별도 섹션이나 별도 카테고리를 만들지 않는다. 일반 할인, 단순 판매 개시, 사소한 스펙 변경처럼 전략적 의미가 약한 항목은 제외한다.
- Google/Amazon은 직접 TV 발표가 아니더라도 TV 화면 경험, 거실 미디어/커머스, 스마트홈 제어와 연결될 수 있는 경우만 포함한다.
- 모든 보고서는 한국어로 작성한다.
- 제목은 `일간 TV 모니터링 리포트`로 한다.
- `AGENTS.md`의 `Report Format Contract` 섹션에 정의된 섹션 순서, 필드명, 비어 있는 섹션 표기 방식을 반드시 따른다.
- `new_features/YYYY-MM-DD.md` 또는 `new_features/YYYY-MM-DD_요약.md`와 `new_features/latest.md`를 모두 갱신한다.
- 신규 발표 확인 사항 또는 간접 서비스에 포함 항목이 있으면, 가장 중요한 1건을 20자 이내 한국어로 요약해 날짜 뒤에 붙인다. 가독성을 위해 접미사에는 공백을 사용할 수 있다. 예: `new_features/2026-05-08_Google TV AI 기능.md`.
- 신규 발표 확인 사항과 간접 서비스가 모두 `해당 없음`이면 기존처럼 `new_features/YYYY-MM-DD.md`를 사용한다.
- 공통 출처 목록 섹션은 만들지 말고, 각 항목 끝에 `출처`를 둔다.
- 확인된 직접 TV/모니터/프로젝터 항목과 포함한 간접 Google/Amazon 항목에는 `관련성`, `중요도`, `인사이트` 아래 `의미:`, `참고할 점:`, `제안:`을 반드시 넣는다.
- 신규 발표가 없으면 `신규 발표 확인 사항` 아래에는 `해당 없음`만 적는다.
- 루머와 유출은 제외하되, 매우 신뢰도 높은 매체의 보도라면 미확인으로 명확히 표기한다.

제한사항:

- `git commit`, `git push`, `gh`, PR 생성 같은 게시 작업은 하지 않는다.
- 리포트 생성 외의 저장소 구조 변경은 하지 않는다.
- 네트워크 이슈나 검증 공백이 있으면 리포트 본문에만 기록하고 종료한다.

리포트 형식이 바뀌지 않았더라도, 확인한 최신 기준 시각과 검색 구간이 반영되도록 필요한 경우 파일을 갱신해.
